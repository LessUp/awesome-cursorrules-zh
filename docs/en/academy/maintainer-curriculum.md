# Maintainer Curriculum

This page describes how a content change should move through the system.

## Update order

1. Update the rule assets or source evidence first
2. Re-sync generated site facts before building docs
3. Revise narrative pages if the thesis or IA changed
4. Re-check research references when the project needs fresh methodological support

## Maintainer checks

- If a page mainly explains why the system exists, it belongs in introduction or architecture.
- If it explains how to operate or adopt the project, it belongs in guides.
- If it provides samples and categories, it belongs in the rules index.

## Publication consistency

Maintainers need to protect three invariants:

1. The Chinese and English IA stay aligned
2. Figures remain legible in both color themes
3. Homepage facts come from the sync script, not manual edits
