# Academy Path

The academy is not a tutorial bucket. It is a role-based learning path that helps readers build the right mental model in the right order.

## Three core paths

### Value path

For tech leads, interviewers, and open source reviewers.

1. Read the [Project Mission](../introduction/mission)
2. Inspect the [System Overview](../architecture/system-overview)
3. Validate through [Research References](../research/references) and the [Rules Index](../rules/)

### Architecture path

For readers who need to understand theme layers, content system boundaries, and publication logic.

1. Start with the [System Overview](../architecture/system-overview)
2. Continue to the [Site Blueprint](../architecture/blueprint)
3. Use [Composition Patterns](../rules/composition-patterns) to confirm that the evidence layer supports the wider narrative

### Adoption path

For maintainers, contributors, and internal DevEx owners.

1. Read the [Maintainer Curriculum](./maintainer-curriculum)
2. Follow the [Team Onboarding](../guides/team-onboarding) steps
3. Return to the [Rules Index](../rules/) to choose the active rule baseline
