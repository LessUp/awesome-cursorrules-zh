# Adoption Path

## Recommended sequence

1. Identify project shape: solo app, team repo, monolith, or multi-package workspace.
2. Choose one general rule plus one primary stack rule.
3. Validate with a real coding cycle before adding more layers.
4. Write recurring project constraints back into the rules.
5. Add lightweight governance through Git and PR review.
