# Project Mission

The mission is not to collect more rules. The mission is to turn AI-assisted coding experience into a **reviewable, explainable, reusable** engineering knowledge system.

## Why this needed a redesign

A repository that only advertises “132+ rules” still reads like a directory. Directories answer “how many”, but not “why this matters”, “how it should be adopted”, or “why the structure is credible”.

The redesign reframes the project into three layers:

1. **Introduction** for thesis and reader framing
2. **Academy** for role-based learning paths and maintainer logic
3. **Rules and research evidence** for validation after the thesis is clear

## Who this serves

- **Interviewers and reviewers** who need a high-signal view of project maturity
- **Senior GitHub developers** who want to see a coherent system, not only collected assets
- **Maintainers and contributors** who need a consistent publishing and governance model
