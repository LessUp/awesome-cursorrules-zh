# 项目使命

这个项目的使命不是“收集更多规则”，而是把 AI 辅助编程中的经验沉淀为**可审阅、可解释、可复用**的工程知识系统。

## 为什么现在要重新定义它

如果仓库只展示“132+ 规则”，高级开发者看到的仍然只是一个目录。目录能回答“有多少”，却不能回答“为什么重要”“如何采用”“为什么这套结构可信”。

这次重构要把项目从规则清单提升为三层价值叙事：

1. **项目导读**：先说明它为什么值得被技术负责人认真审阅。
2. **学院路径**：再把不同角色的阅读顺序、采用节奏和维护责任拆开。
3. **规则与研究证据**：最后用规则索引、组合模式和引用资料为判断提供证据。

## 这套系统服务谁

- **严苛的面试官**：快速判断项目是否具有完整的思想深度与架构表达能力。
- **GitHub 高级开发者**：评估仓库是否只是素材堆积，还是具备可持续演进的内容系统。
- **维护者与贡献者**：在双语、规则资产、页面叙事之间保持一致的发布与治理逻辑。
