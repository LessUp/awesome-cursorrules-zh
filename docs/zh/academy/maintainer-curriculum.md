# 维护者课程

这个页面面向维护者，描述一次完整的内容更新应该如何发生。

## 更新顺序

1. **先更新规则资产或证据来源**：确保 `docs/rules/` 中的权威内容先成立。
2. **再运行站点事实同步**：让首页与配置里的计数、领域覆盖自动更新。
3. **随后修订叙事页面**：如果信息架构或主张发生变化，应同步更新导读、学院和架构页。
4. **最后检查研究引用**：确认新增内容是否需要补充方法论或相关项目引用。

## 维护判断准则

- 新增内容如果只解释“怎么做”，应落在指南层。
- 新增内容如果解释“为什么这样组织”，应落在导读或架构层。
- 新增内容如果只提供样本与分类，应落在规则索引层。

## 发布面的一致性

维护者需要同时守住三件事：

1. 双语目录树保持一致
2. 浅色 / 深色模式下图示可读
3. 首页指标来自同步脚本，而不是人工复制
