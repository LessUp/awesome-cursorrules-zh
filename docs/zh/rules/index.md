# 规则总览

这里是项目的**规则资产地图**。  
如果白皮书、架构页和方法论负责解释“为什么”和“怎么做”，那么本页负责回答“具体有哪些规则可用”。

## 使用建议

1. 如果你还不确定从哪里开始，先去看 [采用路径](../playbook/adoption-path.md)。
2. 如果你想理解站点为何这样组织，先去看 [站点蓝图](../architecture/blueprint.md)。
3. 如果你已经知道自己的技术栈，再回到这里按分类进入。

## 如何阅读这个页面

1. 先定位主技术栈，再判断是否需要安全、数据、基础设施等叠加规则。
2. 不确定规则边界时，优先选择更少的规则组合，跑一轮真实开发任务。
3. 当你开始重复补充某些约束时，把它们写回项目自己的 `.cursorrules`。
