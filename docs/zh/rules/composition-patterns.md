# 规则组合模式

本页帮助你先判断应该验证哪一组规则，再进入具体分类页面。

## 基础组合

- 通用 + 主技术栈
- 先从 [通用规则](/zh/rules/general) 与当前主技术栈页面开始

## 升级组合

- 主技术栈 + 安全 / 数据 / 平台
- 当交付链路变长时，再叠加 [安全](/zh/rules/security)、[数据](/zh/rules/data) 或 [平台](/zh/rules/platform)

## 治理升级信号

- 当团队反复补充同类约束时，把它们沉淀为自定义规则
- 当评审、交付与协作边界开始重复说明时，把组合升级为团队自己的 `.cursorrules`
