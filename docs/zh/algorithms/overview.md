## 规则编排算法的四段式流程

### 1. 规则发现

规则发现从 `rules/` 开始，递归识别每一个 `.cursorrules` 文件。这里不做复杂推断，先保证资产边界清晰。

### 2. 分类归约

系统把所有规则按目录投影为领域分类，并进一步形成 `categoryDistribution`、`coverageBuckets` 与 `qualitySignals`。这一步决定了站点如何描述覆盖广度和知识密度。

### 3. 组合与路径设计

真正体现“规则编排算法”的地方在这里：我们不把规则当作互不相关的模板，而是将其组织为阅读路径、组合模式和升级路径，让读者知道应该如何从单条规则走向系统理解。

### 4. 发布证据

最后一步不是简单输出 Markdown，而是把算法结果映射到首页、系统架构、性能白皮书和参考资料页，形成完整的发布证据链路。

## 为什么这是一种算法说明

- 它有**输入**：规则资产与目录结构。
- 它有**转换**：分类、分桶、组合、叙事路径。
- 它有**输出**：导航、图示、性能指标、参考与演进页面。
- 它可被**验证**：站点测试可以检查页面是否存在、导航是否正确、图示组件是否注册。

## 与系统架构的关系

系统架构回答“层级与边界是什么”，规则编排算法回答“这些层是如何把资产转化为结果的”。二者合在一起，才构成完整的工程白皮书。
