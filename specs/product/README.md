# 产品需求文档 (PRD)

本目录用于存放产品需求文档。

## PRD 模板

```markdown
# [功能名称] 产品需求文档

## 背景
[描述为什么需要这个功能]

## 目标
[描述这个功能要解决的问题]

## 用户故事
作为 [角色]，我希望 [行为]，以便 [价值]

## 功能需求
1. [需求1]
2. [需求2]

## 非功能需求
- 性能要求
- 安全要求
- 兼容性要求

## 验收标准
- [ ] 标准1
- [ ] 标准2

## 时间线
- 预计开始日期
- 预计完成日期
```

## 文档命名规范

- 使用小写字母和连字符
- 格式：`[功能名称]-prd.md`
- 示例：`user-authentication-prd.md`
