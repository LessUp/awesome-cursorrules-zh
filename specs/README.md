# Spec-Driven Development (SDD) 规范

本目录包含项目的规范文档，遵循 Spec-Driven Development 开发范式。

## 目录结构

```
specs/
├── product/     # 产品需求文档 (PRD)
├── rfc/         # 技术设计文档 (RFC)
├── api/         # API 接口定义
├── db/          # 数据库规范
└── testing/     # 测试规范
```

## 开发流程

1. **需求阶段**: 在 `product/` 中定义产品需求
2. **设计阶段**: 在 `rfc/` 中编写技术设计方案
3. **接口定义**: 在 `api/` 中定义 API 接口
4. **数据设计**: 在 `db/` 中定义数据库规范
5. **测试规划**: 在 `testing/` 中编写测试规范

## 文档规范

- 所有文档使用 Markdown 格式
- 文件命名使用小写字母和连字符
- 每个文档应包含：背景、目标、详细说明、验收标准

## 与 AGENTS.md 的关系

AGENTS.md 中定义的开发工作流程以本目录的规范为基础，所有开发活动应遵循本目录中的规范文档。
