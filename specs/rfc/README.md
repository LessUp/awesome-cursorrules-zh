# 技术设计文档 (RFC)

本目录用于存放技术设计文档 (Request for Comments)。

## RFC 模板

```markdown
# [RFC 编号]: [技术方案名称]

## 状态
[草稿/评审中/已批准/已实现/已废弃]

## 摘要
[简要描述方案内容]

## 动机
[为什么需要这个方案]

## 详细设计
[技术实现细节]

### 架构设计
[系统架构图和说明]

### 数据模型
[数据结构设计]

### API 设计
[接口定义]

## 替代方案
[考虑过的其他方案及优缺点]

## 风险与挑战
[可能遇到的问题]

## 实施计划
[分阶段实施步骤]

## 参考文献
[相关资料链接]
```

## 文档命名规范

- 使用 RFC 编号前缀
- 格式：`rfc-[编号]-[名称].md`
- 示例：`rfc-001-vitepress-i18n.md`

## RFC 编号规则

- 编号从 001 开始递增
- 已废弃的 RFC 保留编号但标记状态
