## What You Can Do Here

- Browse the rule library organized by technical domains to quickly locate suitable `.cursorrules`
- Understand the scenarios, tech stacks, and maintenance boundaries for each rule
- Switch between Chinese and English sites to compare the same content
- Reduce the cost of searching through the repository page by page

## How to Choose Rules

- First determine your project's tech stack, then navigate to the corresponding category
- Prioritize rules that are specific enough to be implemented directly, rather than generic templates
- If your project contains multiple technical layers, start from the most core layer
- Read the explanations in the rule file before deciding to copy it to your project root

## Recommended Workflow

- Open the homepage and navigate to the corresponding technical category
- Read the rule description and confirm it aligns with your project goals
- Place `.cursorrules` in your project root directory as a project-level constraint
- Fine-tune the content according to team needs while keeping rules readable and reviewable
- Return to the site and specification docs when changes occur to confirm information consistency

## Project Quality & Standards

- The site uses `zh/` and `en/` as public content sources to avoid duplicate body content
- `specs/` carries project specifications and governance constraints, `docs/` is mainly for non-published content
- Validation, build, and contribution processes remain lightweight, prioritizing trustworthy results
- If you're ready to submit changes, please confirm they comply with existing specifications and maintenance boundaries
