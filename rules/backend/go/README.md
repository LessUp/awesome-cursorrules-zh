# Go 后端开发 .cursorrules 提示文件

## 你可以构建什么

- **高性能后端服务**：利用 Go 的并发特性构建高吞吐量服务
- **微服务架构**：可扩展、可维护的微服务系统
- **云原生应用**：适用于 Kubernetes 部署的容器化应用

## 概要

这个规则文件为 Go 后端项目提供了全面的开发规范。它涵盖了项目结构、并发模式、API 设计和性能优化等方面。

## 核心特性

- 标准项目结构
- 并发模式（goroutine、channel）
- REST / gRPC API 设计
- 数据库和缓存集成
- 容器化部署

## 技术栈

- Go
- REST / gRPC
- PostgreSQL / MySQL
- Redis
- Docker / Kubernetes
